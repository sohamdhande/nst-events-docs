# ADR-026: Mobile application should be student-first

## Status
Accepted

## Context
The vast majority of users will be students looking to attend events.

## Decision
Mobile application should be student-first.

## Consequences
Positive: Students are the primary mobile users. Focused UX leads to higher adoption.
Negative: Admins must use the web app.

## Alternatives Considered
* **Heavy admin mobile app**: Rejected as it compromises the core student experience.

## Future Impact
Ensures the mobile app feels fast and intuitive.
