# ADR-015: Support Event-Based Attendance

## Status
Accepted

## Context
Some events are simple and only require a single check-in.

## Decision
Support Event-Based Attendance.

## Consequences
Positive: Single-session events are quick to set up.
Negative: Requires a flexible schema to handle both types of attendance.

## Alternatives Considered
* **Enforced sessions for all events**: Rejected to preserve good UX for simple events.

## Future Impact
Streamlines the organizer experience.
