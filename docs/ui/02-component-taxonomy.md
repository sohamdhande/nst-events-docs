# Component Taxonomy

Our taxonomy categorizes components strictly by their domain purpose.

## Navigation
Components responsible for moving the user between different contexts (Sidebar, Tabs, Command Palette).

## Events
Surfaces for discovering, viewing, and managing events (Cards, Detail Drawers).

## Attendance
Strictly components handling the check-in lifecycle (QR Renderers, Scanners, Timelines).

## Clubs
Modules for organization identity and roster management.

## Notifications & Communication
Any UI designed to interrupt or inform the user outside their current focus (Toasts, Banners).

## Profiles
User identity and settings surfaces.

## Leaderboard
Merit-based gamification surfaces (Podiums, Badges, Ranks).

## Analytics
Data visualization primitives (Charts, Heatmaps).

## Administration
Platform-level management tools (Audit Viewers, Role Assignment).

## Operations
High-stress, live-event components designed for rapid, distraction-free execution.

## System Components
Base primitives (Buttons, Modals, Empty States, Skeletons) that construct all other domains.

## Forms
Data entry primitives.
