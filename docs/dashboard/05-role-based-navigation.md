# Role-Based Navigation

## Platform Admin
* **Primary Workflows**: System configuration, audit log review, global club creation.
* **Hidden Domains**: None.

## Faculty Admin
* **Primary Workflows**: Campus-wide event approvals, global faculty analytics, creating global events.
* **Hidden Domains**: System audit logs, raw database settings.

## Faculty Mentor
* **Primary Workflows**: Approving events for assigned clubs, creating events, viewing club-specific analytics.
* **Hidden Domains**: Global campus analytics, unassigned clubs.

## Club Admin
* **Primary Workflows**: Creating events, managing rosters, launching Operations Mode, broadcasting announcements.
* **Hidden Domains**: Event Approval queues (cannot approve their own events).

## Core Member
* **Primary Workflows**: Drafting events, assisting in Operations Mode.
* **Permissions**: Cannot broadcast campus-wide announcements or manage the roster.
